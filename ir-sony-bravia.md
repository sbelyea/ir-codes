# IR Codes for the Sony Bravia TV Remote
These codes were captured with an ESP32 device running ESPHome, wired up with an IR Receiver.  The Sony remote is a model RMF-TX520U.  No guarantee the IR codes generated by this device will work on other Sony TVs.  Codes are provided as the raw 

## ESPHome configuration for capturing IR

I heavily referenced the following resources to get this configuration created:
- [Setting up Remote / IR Devices](https://new.esphome.io/guides/setting_up_rmt_devices/)
- [Setting up a Remote / IR Transmitter](https://esphome.io/components/remote_transmitter/)
- [Sensus IR code converter](https://pasthev.github.io/sensus/) (Very helpful for translating pronto output into something that can be fed in to an ESPHome YAML; the actual project on Github is here: https://github.com/pasthev/sensus)

Here's the code block I used for my ESPHome config.
```yaml
esphome:
  name: esp32-00
  friendly_name: esp32-00

esp32:
  board: esp32dev
  framework:
    type: esp-idf

# Enable logging
logger:

# Enable Home Assistant API
api:
  encryption:
    key: "[]" // Removed for privacy

ota:
  - platform: esphome
    password: "[]" // Removed for privacy

wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password
  manual_ip:
    # Set this to the IP of the ESP
    static_ip: 192.168.2.200
    # Set this to the IP address of the router. Often ends with .1
    gateway: 192.168.2.1
    # The subnet of the network. 255.255.255.0 works for most home networks.
    subnet: 255.255.255.0

  # Enable fallback hotspot (captive portal) in case wifi connection fails
  ap:
    ssid: "Esp32-00 Fallback Hotspot"
    password: "wuZKRp6Q52LL"

captive_portal:

// For this, I wired up the "OUT" pin on my IR Receiver to GPIO #15 on my ESP32.  Used an ESP32-WOOX device.
remote_receiver:
  pin: GPIO15
  dump: all
```
